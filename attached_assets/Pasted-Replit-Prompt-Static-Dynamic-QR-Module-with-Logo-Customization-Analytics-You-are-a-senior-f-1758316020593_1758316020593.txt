Replit Prompt — Static & Dynamic QR Module (with Logo, Customization, Analytics)

You are a senior full-stack engineer.
Add a QR Codes module to my 2TalkLink app (Node/Express, React/Tailwind, Supabase/Postgres).
The module supports Static QR and Dynamic QR:

Static QR: encodes a fixed URL or text; no DB, no analytics; downloadable PNG/SVG; customizable colors/size/logo.

Dynamic QR: creates a short link (e.g., /q/:short_id) that redirects to a target URL; target can be edited later; scans are tracked in analytics; customizable PNG/SVG with center logo.

0) Tech & Conventions

Backend: Express.js, Supabase/Postgres, TypeScript preferred.

Frontend: React + Tailwind.

Libraries: qrcode (or qrcode-generator) for QR; ua-parser-js for device; express-rate-limit; zod for validation.

Env: IP_HASH_SALT (random string). If behind Cloudflare, trust CF-Connecting-IP and CF-IPCountry.

1) Database (SQL migrations)

Create tables:

qr_links (for Dynamic QR)

id UUID PRIMARY KEY DEFAULT gen_random_uuid()

user_id UUID NOT NULL

short_id TEXT UNIQUE NOT NULL (7–10 URL-safe chars)

name TEXT NULL (label in UI)

target_url TEXT NOT NULL (absolute http/https)

utm JSONB NULL (e.g., { "utm_source":"qr" })

rules JSONB NULL (optional smart routing; see section 6)

enabled BOOLEAN NOT NULL DEFAULT TRUE

created_at TIMESTAMPTZ DEFAULT now()

updated_at TIMESTAMPTZ DEFAULT now()

Indexes: short_id UNIQUE, (user_id, created_at DESC).

qr_events (scan logs for Dynamic QR)

id BIGSERIAL PRIMARY KEY

qr_id UUID NOT NULL (FK → qr_links.id)

ts TIMESTAMPTZ NOT NULL DEFAULT now()

ip_hash TEXT NULL (SHA256 of IP + salt)

ua TEXT NULL

device TEXT NULL (mobile/desktop/tablet/bot)

country TEXT NULL (from header)

referrer TEXT NULL

landing_host TEXT NULL (hostname used)
Indexes: (qr_id, ts), (device), (country).

2) Short ID & Validation Utilities

Function to generate short_id using URL-safe alphabet (a–z, A–Z, 0–9), length 7 by default; check uniqueness; retry on collision.

Zod schemas for inputs:

target_url must be absolute http:// or https://.

Optional UTM object with string values.

rules JSON structure (see section 6).

3) Public Redirect (Dynamic QR)

Add route:

GET /q/:short_id


Flow:

Lookup qr_links by short_id and enabled = true. If missing → 404 minimal HTML.

Build final URL:

Start with target_url.

Merge UTM (append only keys not present; support a flag “overwrite UTM” later).

Apply rules (section 6). Always ensure a final URL (fallback to target_url).

Log event in qr_events:

qr_id, ts, ip_hash = sha256(client_ip|IP_HASH_SALT),

ua, derived device via ua-parser, country from header, referrer, landing_host = req.hostname.

Respond with 302 redirect to final URL.

Protections: basic rate limit (e.g., 30 req/min/IP) on /q/*; set Cache-Control: no-store.

4) Dynamic QR API (Authenticated)

POST /api/qr
Body: { name?, target_url, utm?, rules? }
Validate; create short_id; insert; return { id, short_id, public_url: https://<host>/q/<short_id> }.

PATCH /api/qr/:id
Update any of: name, target_url, utm, rules, enabled; touch updated_at.

GET /api/qr
Paginated list (page, limit, search by name/target_url).
Include computed total_scans (count) and last_scan (max ts).

GET /api/qr/:id
Return full record + public_url + signed preview of QR image.

DELETE /api/qr/:id
Hard delete or soft delete if you add deleted_at.

5) QR Image Generation (Customizable + Logo) — Static & Dynamic

Create one image endpoint that can serve both:

GET /api/qr/image
  // Static mode:
  //   query: ?mode=static&data=<URL-OR-TEXT>&format=png|svg&size=256|512|1024&margin=2&dark=%23000000&light=%23ffffff&logoUrl=https://...
  //
  // Dynamic mode:
  //   query: ?mode=dynamic&id=<qr_id or short_id>&format=png|svg&size=256|512|1024&margin=2&dark=%23000000&light=%23ffffff&logoUrl=https://...


Behavior:

Use qrcode lib to generate base QR (high error correction level so logo doesn’t break scan).

Customization supported:

format: png or svg (default svg)

size: 256 | 512 | 1024 (default 512)

margin: quiet zone (default 2)

dark & light colors (%23 encoded hex)

logoUrl optional → place center logo scaled to ~20–25% of QR width:

SVG: embed as <image> centered.

PNG: render QR on canvas, fetch logo server-side, draw centered.

Headers: correct Content-Type, Cache-Control: public, max-age=86400.

Notes:

In static mode, data is used directly (no DB).

In dynamic mode, short_id or id resolves to public_url (https://<host>/q/<short_id>).

6) Optional Smart Routing Rules (Dynamic)

Store JSON in qr_links.rules. Example:

{
  "platform": { "ios": "https://example.com/ios", "android": "https://wa.me/123456789" },
  "device": { "desktop": "https://example.com/desktop", "mobile": "https://example.com/mobile" },
  "geo": { "US": "https://example.com/us", "DE": "https://example.com/de" },
  "ab": [ { "weight": 60, "url": "https://example.com/a" }, { "weight": 40, "url": "https://example.com/b" } ],
  "fallback": "https://example.com/default"
}


Precedence: platform → device → geo → ab → fallback.
If a section is missing, skip it. Always return a final URL (else use target_url).

9) Security, Privacy & Performance

No PII: never store raw IP; only ip_hash with salt.

Rate limit: /q/* and /api/qr/*.

Bot hygiene: mark obvious bots via UA → set device = "bot"; optionally exclude from charts via toggle.

Redirect response: Cache-Control: no-store.

Image responses: cache 24h (public, max-age=86400).

Short URL generation: collision-safe.

10) Custom Domains Compatibility

When serving /q/:short_id, record landing_host = req.hostname.

When showing public short URL in UI, use current origin so white-label domains work automatically later.
