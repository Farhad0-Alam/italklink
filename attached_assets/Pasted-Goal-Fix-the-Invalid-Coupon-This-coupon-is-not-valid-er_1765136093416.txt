-------------------------------------

-------------------------------------

    - Coupon database model / table (e.g. coupons, discount_codes).

1.2 Identify which environment we are using (local/test) for:
    - Coupon DB records

Make sure coupon validation is reading from the SAME database where coupons are stored.

-------------------------------------
2) Ensure PREMIUM30 exists and is configured correctly in code/DB
-------------------------------------
2.1 Inspect the coupons table directly (via ORM query or DB browser) and confirm that there is a row with:
    - code: 'PREMIUM30'

-------------------------------------
3) Fix coupon validation function
-------------------------------------
3.1 Open the backend function that validates the coupon code (the one that ultimately produces “Invalid Coupon”).

3.2 Implement/verify the following logic:

    Given input:
      - couponCode (string from request)
      - planId (or priceId) for the selected plan

    Steps:
      - Normalize couponCode:
            const normalizedCode = couponCode.trim().toUpperCase();
        and ensure the DB stores coupon codes in uppercase as well, or compare case-insensitively.

      - Query the coupon record by normalizedCode.
      - If not found:
            return { valid: false, message: 'Coupon not found' };

      - Check status === 'Active'. If not, return message 'Coupon is inactive'.

      - Check startDate <= now and (expiryDate == null || expiryDate >= now).
        If invalid by date, return message 'Coupon is expired or not yet active'.

      - If coupon has applicablePlans:
            - If the selected planId is NOT in coupon.applicablePlans:
                  return { valid: false, message: 'Coupon not valid for this plan' };

      - (Optional) Check usage limits (totalUsageLimit, usesPerUser) if these are implemented.
        If exceeded, return 'Usage limit exceeded'.

      - If all checks pass:
            return { valid: true, couponId, discountType, percentage/amount, message: 'Coupon applied' }.

3.3 Make sure the frontend error banner "Invalid Coupon – This coupon is not valid" is only shown when
     the backend returns valid: false. If valid: true, do NOT show that banner.
