You are a senior full-stack engineer. Modify my 2TalkLink (Node/Express + React + Tailwind + Supabase) app to let users upload an .html file and publish it at a top-level URL: https://2talklink.com/:slug

Key goals
- Users upload a single HTML file from the dashboard and pick a custom slug.
- Public page is served at `/:slug` (no prefix).
- Must NOT break existing app routes (API, auth, builder, SPA paths). Use a reserved-slug allowlist to avoid conflicts.

Data & Storage
- DB table: public_html_pages(
    id uuid pk default gen_random_uuid(),
    user_id uuid not null,
    slug text unique not null,
    storage_path text not null,
    title text null,
    is_public boolean default true,
    created_at timestamptz default now(),
    updated_at timestamptz default now()
  )
- Supabase Storage bucket: `public-html` (public read).
- storage_path pattern: `user_{userId}/{slug}.html`
- Max size 2 MB, MIME must be `text/html`.

Reserved slugs (MUST NOT be user slugs)
[
  "", "api", "auth", "admin", "builder", "dashboard", "login", "register",
  "logout", "static", "assets", "public", "favicon.ico", "robots.txt", "sitemap.xml",
  "_next", "card", "cards", "user", "users", "settings", "docs", "pricing",
  "health", "status", "webhook", "hooks", "oauth", "pay", "stripe", "paypal"
]

Backend (Express)
1) Middleware order:
   - API routes first (e.g., /api/*).
   - Static assets (/assets, /static, /favicon.ico, /robots.txt, /sitemap.xml).
   - Reserved slugs short-circuit to `next()` (to SPA or 404 as usual).
   - Then a dedicated handler for `/:slug` to check DB for public_html_pages.
   - Finally the SPA fallback `app.get('*', sendIndex)`.

2) Route: GET '/:slug'
   - If slug is reserved => next().
   - Query public_html_pages where slug = :slug AND is_public = true.
   - If found:
       Option A (recommended): 301 redirect to the Supabase public URL.
       Option B (stream): Fetch bytes from Supabase and send with headers:
         Content-Type: text/html; charset=utf-8
         Cache-Control: public, max-age=300
         X-Content-Type-Options: nosniff
         Referrer-Policy: strict-origin-when-cross-origin
     Return 404 minimal HTML if not found.
   - If error: 500 JSON.

3) API endpoints (auth required):
   - POST /api/html-pages  (multipart: file, slug, title?)
     * Validate slug regex: ^[a-z0-9-]{3,80}$
     * Reject if slug in reserved list or exists in DB.
     * Verify file size ≤ 2MB and mime `text/html`.
     * Upload to Supabase bucket -> storage_path.
     * Insert row -> return { id, slug, url: `https://domain/${slug}` }.
   - GET /api/html-pages (list current user’s pages, with pagination)
   - PATCH /api/html-pages/:id  (update title, toggle is_public; optional replace file with same validations)
   - DELETE /api/html-pages/:id (delete DB row and storage object)

Frontend (Dashboard)
- New menu: “HTML Pages”
- List table: Title | Slug | Public URL | Created | Actions (Preview, Copy Link, Edit Title, Replace File, Toggle Public, Delete)
- “New HTML Page” modal:
  * Inputs: File (.html only), Slug (auto-slugify), Title (optional)
  * Client validations: size, extension, slug regex, reserved slugs (mirror list)
  * On success: show Copy Link (https://domain/:slug) and Preview.
- Replace file keeps same slug/URL.
- Nice UX: toast notifications, loading states.

Routing Safety
- Ensure `app.get('/:slug')` is defined BEFORE SPA fallback, AFTER API/static.
- `/:slug` handler MUST skip reserved slugs.
- SPA routes (React Router) remain unaffected due to reserved list and order.

Security & Notes
- Do NOT rewrite user HTML; serve as-is.
- Keep CSP off for streamed HTML to avoid breaking user inline scripts; can make configurable later.
- Consider adding view counter behind a feature flag:
  table public_html_hits(slug text, hit_at timestamptz, ip_hash text)
- Add unit tests for slug validator and duplicate slug handling.

Deliverables

- Supabase bucket init & public policy notes.
- Express routes/controllers/middleware order update.
- React dashboard: List + Create/Edit/Replace/Toggle.
- README: env vars, reserved slugs list, deployment notes.

Return
- Full code diffs/files.
