# Project Goal
Build a production-ready "RAG Voice Conversation" system for Talklink using:
- Replit PostgreSQL (with pgvector) as the knowledge base
- OpenAI Realtime API for voice conversation (similar to ChatGPT Voice mode)
- React + TypeScript frontend (Talklink builder)
- Node.js + TypeScript backend with WebSocket for real-time audio

The final result must allow a Talklink page element (Voice Assistant Element) to:
- Start a live voice conversation from the browser (mic permission popup)
- Stream user audio to the backend
- Use RAG over kb_docs table for knowledge-base answers
- Receive real-time transcripts + assistant responses
- (Optional but preferred) stream back synthesized audio to the browser

---

## Stack & Dependencies

1. Backend
   - Node.js + TypeScript
   - Express or Fastify for HTTP API
   - `ws` or `socket.io` for WebSocket server
   - `pg` + `pgvector` for PostgreSQL embeddings
   - `openai` for Realtime API & embeddings

2. Frontend
   - React + TypeScript
   - Existing Talklink UI system (shadcn/ui, Tailwind)
   - WebSocket client for voice streaming
   - Web Audio APIs for mic capture and audio playback

3. Environment variables
   - `OPENAI_API_KEY` (required)
   - `DATABASE_URL` (Replit PostgreSQL connection string)
   - `NODE_ENV=production`

Make sure the project builds and runs on Replit without manual steps.

WebSocket Server for Voice Streaming (Backend)
Create file: src/server/voiceRealtimeServer.ts

Responsibilities:

Start a WebSocket server on path /api/voice/realtime.

Each new WebSocket connection:

Parse query params or first message to get:

orgId

cardId (optional)

sessionId (optional)

Initialize one instance of RealtimeClient per connection.

Client → Server flow:

Client sends messages in JSON envelopes:

{ "type": "audio_chunk", "data": ArrayBufferBase64 }

{ "type": "user_text", "text": string }

{ "type": "stop" }

Server routes:

audio_chunk → realtimeClient.sendUserAudio()

user_text → realtimeClient.sendUserText()

stop → gracefully close realtime and WebSocket.

Server → Client flow:

Forward realtime events as JSON:

{"type":"transcript_partial","text":string}

{"type":"transcript_final","text":string}

{"type":"assistant_delta","text":string}

{"type":"assistant_complete","text":string}

later (if implemented): {"type":"assistant_audio","audioChunk":Base64}

Make sure to:

Handle connection close and cleanup.

Support multiple concurrent connections safely.
