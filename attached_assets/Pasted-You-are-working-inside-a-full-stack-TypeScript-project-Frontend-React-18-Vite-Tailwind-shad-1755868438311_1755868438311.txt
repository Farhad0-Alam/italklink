You are working inside a full-stack TypeScript project:

Frontend: React 18 + Vite + Tailwind + shadcn/ui

Backend: Node.js + TypeScript, Express, Drizzle ORM, PostgreSQL

Existing: chatbot works with text/PDF sources but fails to ingest website URLs.

Goal: add a robust URL → text → chunks → embeddings → pgvector pipeline and expose:

POST /api/ingest { url } — ingest/refresh a public page

POST /api/chat { query } — answer using retrieved context (RAG) with citations

Requirements

Database

Use Postgres with pgvector (1536 dims).

Create table kb_docs(id, url, title, content, content_tokens, meta jsonb, embedding vector(1536), created_at).

Indexes: ivfflat on embedding (cosine), btree on url.

Drizzle

Keep embedding handled via raw SQL (Drizzle schema omits vector type).

Provide db.ts for Pool + drizzle init; schema.ts for kbDocs (minus vector).

Ingestion (URL → text)

Implement fetchReadable(url) using JSDOM + @mozilla/readability to extract main content.

Normalize text (collapse whitespace).

If < 200 chars or readability fails, return a clear error and include a plug-in point for a crawling API (function stub fetchViaCrawler(url) that can be enabled via CRAWLER_API env, but don’t require third-party keys to run).

Chunk by tokens or characters (e.g., ~800–1000 tokens or ~4k chars) with overlap (~10–15%). Provide both implementations; default to character chunks to avoid tiktoken dependency.

Embeddings

Use OpenAI text-embedding-3-small by default (1536 dims).

Batch embed chunks; map to pgvector literal '[f1,f2,...]'.

Upsert

Insert one row per chunk with meta: { chunkIndex }.

Avoid duplicates by (url, content) hash: compute sha256(content) and store in meta.hash; skip if exists. (Add a unique partial index if simple to do; otherwise check in code.)

Retrieval + Answer

POST /api/chat:

Embed query → nearest neighbors (cosine) from kb_docs (topK=5).

Build context by concatenating top chunks with numbered brackets [#1].

Call chat model (env CHAT_MODEL, default “gpt-5” or latest available) with a system prompt: “Answer using ONLY provided context; if unknown, say you don’t know; include short citations like [#1].”

Return { answer, sources:[{id: "#1", url, title, score}] }.

Routes

SQL (run once):

CREATE EXTENSION IF NOT EXISTS vector;

CREATE TABLE IF NOT EXISTS kb_docs (
  id BIGSERIAL PRIMARY KEY,
  url TEXT NOT NULL,
  title TEXT,
  content TEXT NOT NULL,
  content_tokens INT,
  meta JSONB,
  embedding VECTOR(1536),
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX IF NOT EXISTS kb_docs_embedding_idx
ON kb_docs USING ivfflat (embedding vector_cosine_ops) WITH (lists = 100);

CREATE INDEX IF NOT EXISTS kb_docs_url_idx ON kb_docs(url);
