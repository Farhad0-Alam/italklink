Goal: Make the “Profile” element in the Page Builder fully dynamic and feature-complete, exactly like the main Profile section (hero/profile header), including Shape Divider and all styling options.

Scope:
- Work ONLY on the Page Builder profile-related components, page element schema, and their settings panels.
- Do NOT touch billing, Stripe, coupons, shop_cart, or unrelated pages.

------------------------------------------------
1) Understand current implementations
------------------------------------------------
1.1 Find:
    - The main Profile / Hero section component that already works correctly (shows name, title, company, avatar, cover, shape divider, etc.).
    - The Page Builder “Profile” element component that users can drag/drop into the page.
    - The schema / JSON config used to store element settings (for example: pageElements, elementConfig, profileSettings).

1.2 Compare them and list which fields exist in the main Profile section but are missing in the Profile element.

------------------------------------------------
2) Clone full Profile data into the Profile element
------------------------------------------------
Make the Page Builder “Profile” element support the same data + options as the main Profile section:

2.1 Data / text fields:
    - name
    - title
    - company
    - subtitle / description (if exists)
    - visibility flags for each field (show/hide)

2.2 Image fields:
    - profilePhoto
    - coverImage
    - coverOverlay options if present (overlay color, opacity, blur, etc.)

2.3 Styling & layout:
    - profile image size/shape (circle, rounded, etc.)
    - border, shadow, and animation toggles
    - alignment (left/center)
    - spacing / padding options

2.4 Typography:
    - font styles for name/title/company (font family, size, weight, color)

For each Profile element instance, store these values in its own element config so multiple Profile elements can have different styles and content.

Default behavior:
- When a new Profile element is added, pre-fill its fields from the main card/profile data (user’s name, title, company, photo, cover).
- After that, changes to that element should be independent (editing one Profile element must NOT update others automatically).

------------------------------------------------
3) Shape Divider support (currently missing)
------------------------------------------------
Right now Shape Divider from the main Profile section is not showing on the Profile element. Fix this:

3.1 Reuse the existing Shape Divider implementation:
    - top / bottom position
    - shape type (wave, tilt, etc.)
    - color
    - height / opacity
    - flip / invert options if available

3.2 Add Shape Divider settings to the Profile element config:
    - profile.shapeDivider.enabled
    - profile.shapeDivider.position ('top' | 'bottom' | 'both')
    - profile.shapeDivider.type
    - profile.shapeDivider.color
    - profile.shapeDivider.height / intensity

3.3 Render the Shape Divider inside the Profile element exactly like the main section so that it visually matches the reference (second image).

3.4 Ensure each Profile element can have its own Shape Divider settings (fully independent per element).

------------------------------------------------
4) Settings panel & UX
------------------------------------------------
4.1 In the Page Builder settings sidebar for the Profile element:
    - Add controls for all new fields (text, images, typography, cover styling, shape divider).
    - Group them logically (Content, Images, Typography, Shape Divider).

4.2 Make sure changes in the sidebar instantly update only the selected Profile element on the canvas.

------------------------------------------------
5) Backwards compatibility & testing
------------------------------------------------
5.1 Preserve existing pages:
    - Existing Profile elements should keep working.
    - For older elements, populate missing fields with sensible defaults from the main profile data.

5.2 Manual tests:
    - Add a new Profile element to a page: it should auto-fill with user profile data.
    - Change text, photo, and styles: only that element updates.
    - Enable Shape Divider and verify it renders like the main Profile section.
    - Add two different Profile elements on the same page and confirm they work independently.

Rules:
- Do NOT modify non-profile elements (buttons, galleries, etc.).
- Do NOT touch billing/checkout-related code.
- Keep changes minimal and focused on making the Profile element feature-complete and fully dynamic.
