অপশন A: "Force Login/Signup before Checkout" (Simple & Safe)

লজিক (UX Flow):

ইউজার Plan selection পেজে আসে

Subscribe Now ক্লিক করলে প্রথমে check করবে সে logged-in কি না

যদি logged-in না থাকে → তাকে signup / login পেজে পাঠাও, সাথে plan info carry করো

সাইনআপ/লগইন শেষ → আবার redirect করে ওই Plan নিয়ে checkout session তৈরি করো

Frontend (React pseudo-code):
const handleSubscribe = (planId: string, billingPeriod: 'monthly' | 'yearly') => {
  if (!currentUser) {
    // plan info sotre kore redirect
    const params = new URLSearchParams({ planId, billingPeriod });
    localStorage.setItem('pendingPlan', JSON.stringify({ planId, billingPeriod }));
    navigate(`/auth/signup?${params.toString()}`);
    return;
  }

  // logged-in হলে সরাসরি checkout API call
  createCheckoutSession(planId, billingPeriod);
};


Signup/Login সফল হলে:

useEffect(() => {
  if (currentUser) {
    const pending = localStorage.getItem('pendingPlan');
    if (pending) {
      const { planId, billingPeriod } = JSON.parse(pending);
      localStorage.removeItem('pendingPlan');
      createCheckoutSession(planId, billingPeriod);
    }
  }
}, [currentUser]);

/api/billing/create-checkout-session এ auth middleware থাকবে:

অপশন B: "Guest Checkout + Auto Account Creation" (Advanced)

যদি তুমি চাই যে নতুন ইউজার account বানানোর আগেই pay করতে পারবে, তখন:

Plan card–এ একটি email field/step রাখো

Guest হিসাবে checkout শুরু

Backend–এ /api/billing/create-checkout-session কল করলে:

যদি req.user না থাকে কিন্তু email পেল =>

নতুন user create করো (inactive/guest)

Stripe customer create করো

Stripe checkout session বানাও

Payment success হলে webhook–এ:

user-এর account activate + password set করার জন্য magic-link ইমেইল পাঠাও

এটা একটু বেশি কাজ, কিন্তু খুব প্রফেশনাল সিস্টেম হয়।
