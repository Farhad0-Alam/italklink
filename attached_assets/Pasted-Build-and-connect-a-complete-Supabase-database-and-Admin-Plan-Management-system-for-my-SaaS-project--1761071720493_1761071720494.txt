Build and connect a complete Supabase database and Admin Plan Management system for my SaaS project 2TalkLink.

🧩 Tech Stack:

React 19 + Tailwind CSS + shadcn/ui

Node.js + TypeScript backend

Supabase (PostgreSQL) for database

🎯 Objectives:

Create the following Supabase tables with correct relationships and RLS (Row Level Security):

users → stores user_id, email, name, avatar_url, role, created_at

digital_cards → linked to users (FK user_id), includes name, title, company, phone, email, website, profile_image_url, social_links (jsonb), video_intro_url, custom_actions (jsonb), theme, is_public, created_at, updated_at

bios → linked to users (FK user_id), includes role, company, bio, tags (jsonb), created_at

connections → logs card scans/clicks (user_id, target_user_id, platform, clicked_at)

plans → plan_name, plan_type (Free/Paid), frequency (Monthly/Yearly), price, features (jsonb), currency, trial_days, card_limit, created_at

subscriptions → user_id (FK), plan_id (FK), status, start_date, end_date, next_billing_date, created_at

analytics → card_id, user_id, page_views, link_clicks, last_visited_at

Use Supabase JS client for all CRUD operations.

Automatically connect database keys using environment variables:

SUPABASE_URL

SUPABASE_KEY

Create a fully functional Admin Dashboard → “Plan Management” page that includes:

Add New Plan (modal or drawer form)

Edit Existing Plan

Delete Plan

Display all plans in a modern responsive table with search/filter

Dynamic frequency (Monthly/Yearly) switch logic

Custom feature checklist (CRM, QR Code, Chatbot, Analytics, etc.)

Auto-sync with Supabase plans table

In the User Dashboard, show available plans dynamically with:

Plan features and limits (from Supabase)

“Upgrade” button connected to Stripe for paid plans

Real-time subscription status (using Supabase Realtime API)

Implement Row Level Security (RLS) so:

Each user can access only their own digital_cards, bios, connections, and analytics data.

Admins (role = admin) can manage all plans and subscriptions.

Deploy all database-connected functions to production and test:

Plan CRUD

Card CRUD

Subscription updates

Analytics tracking

💅 UI/UX Guidelines:

Follow shadcn/ui component structure (Card, Table, Modal, Input, Button).

Use Tailwind colors matching 2TalkLink’s orange branding.

Use font: Inter or Plus Jakarta Sans.

Responsive, clean, and minimal admin layout.

Connected Supabase schema (SQL)
